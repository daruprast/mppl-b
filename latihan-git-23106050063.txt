3. Apa yang dimaksud dengan WIP atau Work In Progress di dalam kanban
    - WIP adalah jumlah pekerjaan yang sedang dilakukan oleh tim dalam satu waktu.
    - WIP perlu dibatasi agar tidak terlalu banyak pekerjaan berjalan sekaligus yang dapat menyebabkan overload dan menurunkan efisiensi
    - Tujuannya adalah meningkatkan fokus, mempercepat penyelesaian tugas, serta menjaga alur kerja tetap lancar
4. Sebutkan dan jelaskan jenis version control system yang digunakan dalam pengembangan perangkat lunak
    - Centralized Version Control System : Sistem terkontrol versi terpusat seperti SVN atau CVS, dimana semua versi kode tersimpan dalam satu server pusat dan pengembang harus terhubung ke server untuk bekerja
    - Distributed Version Control System : Seperti GIT dan Mercurial, setiap pengembang menyimpan salinan lengkap dari seluruh repositori. DVCS lebih fleksible karena memungkinkan pengembangan offline dan lebih aman terhadap kerusakan server pusat. GIT adalah DVCS yang paling sering digunakan saat ini
    - Local Version Control System :Seperti RCS, Menyimpan versi file secara lokal yang biasanya hanya digunakan oleh satu developer. Cocok untuk proyek pribadi berskala kecil yang jarang digunakan dalam pengembangan tim
